The Baics of the Web & HTML - Lesson 1

The world wide web is made up of HTML documents that can be connected via hyperlinks.

The main pieces of the web:

  • computer user
  • computer & browser
  • the internet
  • servers

HTML

HTML documents are the heart of the web and are made up of text, content, the HTML mark-up, references to other documents and links

TAGS

Tags are used to mark-up HTML documents for formating. A tag consists of an opening tag and a closing tag and the whole tag is called an element.

Computers are stupid

Computers are very dumb machines, they will only do what the programmer tells the machine what to do. Because of this, programmers need to be precise on instructions and be sensative to typos in the markup

Links

Links are used to connect an HTML document to any other document on the web. In the previous sentence, web is highlighted in blue and links to www.espn.com.

Inline tags vs Block tags

Inline tags just end a sentence where you put the br tag. Block tags create an invisable box around the text where the box can be formatted Block tags are the Paragraph tags

The structure of an HTML page - lesson 2

A website is like a structure of a house

  • HTML is the structure (rooms)
  • CSS is the style of the house (color, texture)
  • Javascript are the interactive components of house (remote control)

HTML defines the structure and content of a page. Using tags to create elements for the page creates a DOM, or tree-like structure of the page with HTML.

CSS styles the elements of HTML and uses syntax and rules to create style for the page

Everything on a webpage is a box. Boxify page using pencil first before setting up basic structure of the page with div tags. You dont have to put contents in tags at the beginning, just put the tags to start and fill in later. Label div tags with classes for easy styling later. One of the benefits of giving div tags classes is that when you go to style the HTML in CSS, you can style by class name, saving time by not having to style each HTML tag individually. This not only saves time, it also avoids repetition by not having to style each similar div tag.

Lesson 3 - Styling with CSS

CSS is used to style the elements of the HTML of a webpage (see lesson 2). CSS stands for Cascading Style Sheets and there are two key features of CSS to remember:

  • Cascading means that the most specific rule applies to all elements of a page. Think of federal laws down to city laws. The city law would applied federally as well.
  • The second key features is the idea of inhertiance--the idea that styling an element not only applies to that element, but also to its descendants. It uses the document tree and the descendant elements only inherit text properties.
To style in CSS, start with either a html tag, or a html element class, which is how CSS controls how the HTML tags look. For reference, use the CSS reference for attributes and to check CSS code--this is found on the Mozilla Developer Site Network.

The box model revisited

The box model concept is used to position boxes on a web page and consists of 4 properties:

  • Margin--space between boxes on a page and is transparent, you ca't see it
  • Border--basically the outline of a box
  • Padding--clears an area around the center of the box
  • Content--the center of the box

Box sizing & positioning

Boxes can be sized in three ways: size by pixils, size by using a percent, or by using "box-sizing: border-box" for every element. Box positioning for divs by default takes up the whole width of the page. In order to have divs appear next to eachother (like an image and text next to it), use the "display: flex" rule.

Always use the process of code, test & refine when using HTML & CSS and always use the validators to check code for errors
